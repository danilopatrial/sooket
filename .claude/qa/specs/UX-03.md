## What this tests
Verifies that all delete/remove actions require a two-step confirmation before executing — preventing accidental data loss from a single misclick.

## Steps — delete workflow (dashboard)

1. Navigate to the workflow dashboard (`/workflow`)
2. Click the Trash icon on an inactive workflow card
3. Verify the inline confirmation appears: **Delete?** with **Yes** and **No** buttons (the Trash icon is replaced)
4. Click **No** — verify the workflow list returns to its normal state (no deletion)
5. Click the Trash icon again, then click **Yes** — verify the workflow is deleted

## Steps — delete API key (Config → API Keys)

6. Navigate to Config → API Keys tab
7. Click the delete (×) button on a key row
8. Verify the button changes to a **Confirm** / **Cancel** two-button inline confirmation
9. Click **Cancel** — key remains in the list
10. Click the delete button again → **Confirm** — key is deleted

## Steps — delete workflow (Config → General)

11. Navigate to Config → General tab for an inactive workflow
12. Find the **Delete Workflow** section
13. Click the initial delete button — verify it expands to show a confirmation step (e.g. "Are you sure?" with a confirm and cancel button)
14. Click cancel — no deletion
15. Click delete → confirm — workflow is deleted

## Steps — delete variable (Config → Variables)

16. Navigate to Config → Variables tab
17. Click the delete (×) button next to a variable
18. Verify some form of confirmation (inline or modal) before the DELETE request is sent
19. Cancel to preserve the variable

## Steps — import workflow

20. Attempt to import a JSON workflow file on the canvas
21. Verify a confirmation dialog appears before the import replaces the current canvas
22. Dismiss — canvas unchanged

## Expected result
- No destructive action executes on a single click; all require a second explicit confirmation
- Confirmation UI is inline (not a full-page dialog) for dashboard and config list actions
- Cancel always restores the previous state
- Loading state ("Deleting…") appears while the operation is in progress

## Failure indicators
- Clicking the Trash icon immediately deletes a workflow without confirmation
- API key is deleted on first click without "Confirm"
- Cancel button is absent from any confirmation step

## Severity rationale
High: accidental deletion of a workflow or API key is irreversible; a two-click guard is standard practice for preventing data loss.
