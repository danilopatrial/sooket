## What this tests
Verifies that very long workflow names and slugs are truncated with ellipsis on the dashboard and in the canvas header — without causing horizontal overflow, overlapping elements, or broken layouts.

## Steps — dashboard list truncation

1. Create a workflow with a very long name (e.g. 100+ characters):
   ```
   This Is A Very Long Workflow Name That Should Be Truncated When It Exceeds The Available Width In The List Card
   ```
2. Navigate to the dashboard (`/workflow`)
3. Verify the long name fits within its card column — no text extends beyond the card boundary
4. Verify the name shows an ellipsis (`…`) at the truncation point
5. Hover over the name — verify the full name is accessible (either via tooltip or the link still navigates correctly)
6. Verify the **Active**/**Inactive** badge and the **New Workflow** button are not pushed out of position

## Steps — slug truncation

7. Note that workflow slugs are 10-character nanoids (short) — they shouldn't normally truncate
8. Verify the slug row also uses `truncate` so that if a long slug is somehow present, it doesn't overflow

## Steps — canvas header truncation

9. Navigate to the canvas for the long-named workflow
10. Inspect the workflow name shown in the canvas toolbar — verify it uses `truncate` or `min-w-0` to prevent the name from overflowing the header

## Steps — node subtitle truncation

11. Inspect node header subtitles on the canvas (e.g. Anthropic model name, Concat separator preview)
12. Verify that long dynamic values in node subtitles are also truncated — nodes don't expand horizontally due to long text

## Expected result
- Workflow name column uses `truncate` (CSS `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`)
- Slug column uses `truncate`
- Container uses `min-w-0` so `truncate` works correctly inside a flex row
- Layout elements (badge, buttons) remain in their correct positions regardless of name length

## Failure indicators
- Long name extends beyond the card and overlaps the badge or button
- Long name causes the workflow list to scroll horizontally
- Name is cut off mid-character instead of showing ellipsis

## Severity rationale
Low: name truncation is a polish requirement; broken layouts would be immediately visible and affect usability.

## Source reference
`components/workflow/WorkflowList.tsx` line 73 (`text-sm font-medium truncate` on name), line 74 (`text-xs ... font-mono truncate` on slug), line 72 (`space-y-0.5 min-w-0 mr-4` — `min-w-0` enables truncation in flex context).
