## What this tests
Verifies that the trash drop zone in the canvas bottom-right corner is invisible at rest, appears when the user starts dragging a node, and turns red when the dragged node is hovering over it.

## Steps — trash zone invisible at rest

1. Open a canvas editor with several nodes
2. Verify the bottom-right corner is empty — no trash icon or "Drop to delete" text is visible
3. The zone has `opacity-0` at rest (CSS transition, fully invisible)

## Steps — trash zone appears during drag

4. Click and hold (start dragging) any non-Input node
5. Verify a trash zone **appears** in the bottom-right corner with:
   - Semi-transparent dark background (`bg-[#1e1e1e]`)
   - Trash can SVG icon
   - Text: **"Drop to delete"**
   - Reduced opacity (~70%) — visible but not distracting
6. While still dragging, move the mouse away from the trash zone — zone stays visible but dimmed

## Steps — red highlight when hovering over trash

7. While dragging a node, move it over the trash zone in the bottom-right corner
8. Verify the zone changes to:
   - Red background: `bg-red-500/20`
   - Red border: `border-red-500/60`
   - Red text: `text-red-400`
   - Slightly larger scale (`scale-110`)
   - Text changes to: **"Release to delete"**
9. Release the mouse button while over the trash zone — verify the node is deleted

## Steps — releasing outside the trash zone

10. Drag a node toward the trash zone but release it outside the zone
11. Verify the node is NOT deleted — it stays in its new drop position
12. Verify the trash zone fades back to invisible

## Steps — Input node cannot be dragged to trash

13. Attempt to drag the Input node (the default entry node) to the trash zone
14. Verify it cannot be deleted (the Input node is protected — confirmed by release not deleting it)

## Expected result
- At rest: trash zone invisible (`opacity-0`)
- During drag: visible with "Drop to delete" text
- Over trash zone: red highlight, "Release to delete" text, `scale-110`
- Release over zone: node deleted
- Release outside zone: node moves to new position, not deleted

## Failure indicators
- Trash zone always visible (not hidden when not dragging)
- No visual change when hovering over the trash zone during drag
- Releasing outside the zone deletes the node anyway
- The zone has `pointer-events` that interfere with node dragging

## Severity rationale
Low: the trash zone is a convenient but optional delete method; keyboard Delete and right-click context menu are alternatives.
