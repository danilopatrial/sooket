## What this tests
Verifies that every copy-to-clipboard button shows immediate visual feedback — the icon changes to a green checkmark for ~2 seconds and a success toast appears — so users know the copy succeeded.

## Steps — Account page management key copy

1. Navigate to `http://localhost:3000/account`
2. Click the **Copy** button next to the management API key
3. Verify immediately:
   - The Copy icon (`Copy`) changes to a green **Check** icon (`Check h-4 w-4 text-green-500`)
   - A green success toast appears: **"API key copied to clipboard"**
4. After ~2 seconds, verify the icon reverts to the Copy icon (feedback is temporary)
5. Verify the clipboard actually contains the key value (paste into a text field)

## Steps — Config API Keys copy (reveal banner)

6. Navigate to Config → API Keys; create a new key
7. In the one-time reveal banner, click the copy button
8. Verify: Copy icon → Check icon (green), toast with descriptive message
9. After 2 seconds, icon reverts

## Steps — Management key in General Config

10. Navigate to Config → General tab; find the management API key section
11. Click the copy button — same visual feedback: Check icon + success toast

## Steps — Eye toggle works alongside copy

12. On the account page, click the **Eye** icon to reveal the key
13. Verify the key becomes visible (not masked)
14. Click **Eye** again — key is masked
15. Both Eye and Copy buttons are independently functional

## Expected result
- Every copy button: icon switches from `Copy` → `Check` (green) for ~2 seconds on click
- Toast with descriptive message appears simultaneously
- After ~2 seconds, icon reverts to `Copy` (state reset via `setTimeout`)
- Clipboard actually contains the copied value

## Failure indicators
- Copy button shows no visual change after clicking
- Check icon appears but does not revert after 2 seconds
- Toast message is absent or generic (e.g. "Copied")
- Clipboard is empty after clicking copy

## Severity rationale
Medium: without visual feedback, users don't know if the copy succeeded — they may paste incomplete or wrong values.
