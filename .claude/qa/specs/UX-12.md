## What this tests
Verifies that the canvas status bar (bottom strip) shows the current number of nodes on the canvas, updating in real time as nodes are added or removed.

## Steps — status bar presence

1. Open any canvas editor
2. Locate the status bar at the very bottom of the canvas (below the React Flow viewport, above nothing — it's a fixed `h-9` strip with a dark background)
3. Verify it is always visible and contains:
   - **+**, **−**, **fit** zoom control buttons
   - A small **emerald green dot** followed by the node count: e.g. **"3 nodes"**
   - The workflow slug: **"workflow: <slug>"**
   - The API endpoint: **"endpoint: POST .../v1/chat"**

## Steps — node count updates dynamically

4. Note the current node count (e.g. "2 nodes" for Input + Output)
5. Drag a new node from the sidebar onto the canvas
6. Verify the count immediately increments to **"3 nodes"** (or appropriate number)
7. Delete a node (press Delete or use the trash zone)
8. Verify the count decrements to **"2 nodes"**

## Steps — singular/plural form

9. Reduce nodes to exactly 1 (if possible — normally there's always at least the Input node)
10. Verify the text reads **"1 node"** (not "1 nodes")
11. Add another node — verify **"2 nodes"**

## Steps — zoom controls

12. Click **+** in the status bar — verify the canvas zooms in
13. Click **−** — verify the canvas zooms out
14. Click **fit** — verify the canvas fits all nodes in view with a smooth animation

## Expected result
- Status bar always visible at the bottom of the canvas
- Node count shows the total number of React Flow nodes (including Input and Output nodes)
- Count uses correct singular/plural: "1 node" vs "N nodes"
- Emerald dot visually precedes the count
- Count updates immediately when nodes are added or removed

## Failure indicators
- Status bar not visible (hidden below the canvas or behind the Debug panel)
- Node count does not update when nodes are added/removed
- Count shows "1 nodes" instead of "1 node" (missing singular form)
- Emerald dot is absent

## Severity rationale
Low: the node count is an informational display; its absence doesn't break functionality but removes useful at-a-glance feedback.

## Source reference
`components/canvas/WorkflowCanvas.tsx` lines 1325-1354 (status bar: `h-9 border-t`, zoom buttons, node count `{nodes.length} {nodes.length === 1 ? "node" : "nodes"}` with emerald dot, slug and endpoint display).
