## What this tests
Verifies that scrollable content areas (Config panel, Debug panel, node sidebar) are properly contained — no content overflows its bounding box, no unwanted scrollbars appear on the page body, and scrollable sections scroll independently.

## Steps — Config panel scrolling

1. Navigate to `/workflow/<slug>/config`
2. The config page has `overflow-y-auto` — verify the tab content scrolls vertically when there are many items (e.g. many API keys or access list entries)
3. Verify scrolling the config content does NOT scroll the page body or the tab bar above
4. Verify no horizontal scrollbar appears in the config panel at standard viewport sizes

## Steps — Canvas body overflow

5. Navigate to the canvas editor
6. Pan and zoom the canvas — verify no content from the canvas bleeds outside the canvas area
7. Verify the canvas container has `overflow-hidden` — node labels or handles do not appear outside the canvas bounds
8. Verify the sidebar (if open) has its own scroll if it overflows, without affecting the canvas

## Steps — Debug panel scrolling

9. Open the Debug panel; run a request with many node traces
10. Verify the trace list scrolls independently within the panel
11. Resize the Debug panel to minimum height — verify traces remain in a scrollable area, not overflowing into the canvas
12. Verify the Debug panel does not cause the page to grow taller than the viewport

## Steps — Node sidebar scrolling

13. Open the node sidebar; if nodes overflow the sidebar height, verify the sidebar scrolls independently
14. Verify scrolling the sidebar does not scroll the canvas

## Steps — overflow on mobile

15. At 375px viewport width, verify no horizontal scrollbar on any page

## Expected result
- Config panel: `overflow-y-auto` on the container; content scrolls within; body does not scroll
- Canvas: `overflow-hidden` prevents bleed of canvas content
- Debug panel: independent scroll for traces
- No page-level horizontal scrollbars on any page

## Failure indicators
- Scrolling config content also scrolls the browser window
- Canvas content (node edges, labels) appears outside the canvas boundary
- Debug panel trace list extends beyond the panel bounds and is unscrollable
- A horizontal scrollbar appears on any page at standard viewport widths

## Severity rationale
Medium: scroll bleed causes content to overlap other UI elements, making them inaccessible or visually broken.
