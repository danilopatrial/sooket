## What this tests
Verifies that non-obvious UI controls have tooltips (`title` attributes) or visible labels that explain their function — particularly icon-only buttons, toggle switches, and configuration fields with technical terms.

## Steps — icon-only buttons have title attributes

1. On the workflow dashboard, hover over the Trash icon on a workflow card
2. Verify a tooltip appears: **"Delete workflow"** (or similar)

3. On the canvas, hover over the expand (Maximize2) button on a VarField textarea
4. Verify tooltip: **"Expand editor"**

5. In Config → API Keys, hover over the stats (bar chart) icon button on a key row
6. Verify tooltip: **"View usage stats"** (or similar)

## Steps — technical fields have explanatory hints

7. In the Anthropic node, verify the Temperature label includes a descriptive sub-label: **"creative vs precise"** with "precise" / "creative" end labels on the slider
8. In the PromptCompression node, verify the **Target Words** field has a sub-label: **"optional"**
9. In the Auth Validator node, verify the **Token source** field has a hint below: **"header to read the token from · connect to set dynamically"**

## Steps — toggle labels are self-explanatory

10. In the Default Fallback toggle (Router / Language Detect nodes), verify the label **"Default fallback"** appears next to the toggle
11. In the Semantic Cache node, verify the similarity threshold slider has end labels: **"loose"** (left) and **"strict"** (right)

## Steps — placeholder text provides examples

12. In the VarField for a HTTP Request URL, verify placeholder: **"https://api.example.com/endpoint"**
13. In a pattern field for the Content Guardrail, verify placeholder: **"word, phrase, another"**
14. In a regex field for Regex Replace, verify placeholder: **"e.g. \d+ or [aeiou]"**

## Expected result
- All icon-only buttons have `title` attributes for tooltip on hover
- Technical slider controls have contextual end-labels (e.g. "loose"/"strict", "precise"/"creative")
- Field hints and placeholder text provide concrete examples for technical inputs
- Toggle labels are visible next to their toggle switches

## Failure indicators
- Trash/edit/copy icon buttons have no tooltip (hovering shows nothing)
- Temperature slider has no "precise"/"creative" labels
- Similarity threshold has no "loose"/"strict" end labels
- VarField placeholders are empty or say "Enter value..."

## Severity rationale
Low: missing tooltips reduce discoverability for new users; they don't break functionality.

`components/workflow/WorkflowList.tsx` line 113 (`title="Delete workflow"` on Trash button); node component files — tooltips via `title="Expand editor"` on Maximize2 buttons; slider end-labels and field hints throughout `components/canvas/nodes/`.
