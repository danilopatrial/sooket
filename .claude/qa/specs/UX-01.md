## What this tests
Verifies that a first-time user can discover and complete the core workflow: create a new workflow, add nodes, wire them together, and send a debug test request — all without external documentation.

## Prerequisites
- App is running at http://localhost:3000
- No workflows exist (or use a new browser profile)

## Steps — empty state discoverability

1. Navigate to `http://localhost:3000`; verify redirect to `/workflow`
2. Verify the empty state shows helpful text: **"No workflows yet."** and **"Create your first workflow to get started."**
3. Verify the **New Workflow** button is prominently visible in the header area

## Steps — create a workflow

4. Click **New Workflow**
5. Verify the button shows **Creating…** and is disabled (loading state feedback)
6. Verify automatic navigation to the canvas editor (`/workflow/<slug>`)

## Steps — add nodes to the canvas

7. On the canvas, notice the node sidebar (or the sidebar toggle icon)
8. Click to open the sidebar; verify node categories and names are visible
9. Drag a **Text** node onto the canvas — verify it appears at the drop location
10. Connect the Text node's output to the **Input** node (or the Output node) — verify the edge snaps correctly

## Steps — open and use the Debug panel

11. Locate the **Debug** button in the canvas toolbar (bug/beaker icon)
12. Click it — verify the Debug panel opens with a JSON body editor and a **Run** button
13. Enter a test body: `{"message": "hello"}`
14. Click **Run** — verify execution completes and a result is shown in the panel
15. Verify the result is readable (no raw error objects, no blank panel)

## Expected result
- Empty state guides new users toward creating their first workflow
- New Workflow button is easily discoverable with clear loading feedback
- Canvas provides sufficient affordances to add and connect nodes without documentation
- Debug panel is accessible and produces a visible result on first run

## Failure indicators
- Empty state shows no guidance (blank page)
- New Workflow button is hidden or unlabeled
- Canvas has no visual cues for adding nodes (sidebar not discoverable)
- Debug panel run produces no visible result or shows a blank state
- First-time user cannot complete a full test without reading external docs

## Severity rationale
High: if new users cannot complete the core flow without documentation, the product is functionally inaccessible to its primary audience.
