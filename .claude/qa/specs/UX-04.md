## What this tests
Verifies that success and error feedback is immediate, descriptive, and uses the appropriate channel (toast for non-blocking feedback, inline error for blocking states).

## Steps — success toast is descriptive

1. Navigate to Config → Provider Keys; save a new Anthropic key
2. Verify a **green success toast** appears promptly (within 1 second) with: **"Anthropic key saved"** — specific, not generic ("Saved" or "OK")
3. Remove the key — toast: **"Anthropic key removed"**
4. Copy an API key — toast: **"API key copied"**

## Steps — error toast is descriptive

5. Disconnect the network (DevTools → Network → Offline); click **New Workflow**
6. Verify a **red error toast** appears: **"Failed to create workflow"** — not "Error" alone
7. Re-enable the network

## Steps — inline errors for blocking states

8. On the dashboard, trigger a delete error (e.g. force-delete an active workflow via DevTools)
9. Verify the error appears as an **inline banner** in the list with an `AlertTriangle` icon and the specific server error message (e.g. `"Cannot delete an active workflow"`)
10. The error is visible without dismissing; the user can read it and correct the action

## Steps — feedback timing

11. Save a provider key; measure time from click to toast appearance — should be < 2 seconds on localhost
12. Verify the canvas "Saved" indicator appears within ~1 second of stopping canvas edits (800ms debounce + save time)

## Steps — no false positives

13. Verify success toasts only appear on actual success (not on loading or error states)
14. Verify error toasts only appear on actual failures

## Expected result
- Success toasts: green, appear within 1–2 seconds, include the specific item name when applicable
- Error toasts: red, appear immediately on failure, include a human-readable error message
- Inline errors: used when the error is persistent and blocks the user's workflow (not transient)
- No generic "Success" or "Error" messages — every toast identifies what succeeded or failed

## Failure indicators
- Toast appears but with generic message "Done" or "Error occurred"
- Success toast on a network failure
- Error is silently swallowed with no feedback
- Feedback takes > 5 seconds to appear

## Severity rationale
Medium: vague or missing feedback causes users to retry actions unnecessarily or not understand what failed.
