## What this tests
Verifies that switching between Config tabs updates the URL with `?tab=<id>`, and that refreshing the page with a `?tab=` query param restores the correct tab ‚Äî making config URLs bookmarkable and shareable.

## Steps ‚Äî tab switching updates URL

1. Navigate to `/workflow/<slug>/config` (opens on the **general** tab by default)
2. Click the **API Keys** tab
3. Verify the URL changes to `/workflow/<slug>/config?tab=api-keys` without a full page reload (`router.replace` is used ‚Äî no navigation flash)
4. Click **Variables** ‚Äî URL becomes `?tab=variables`
5. Click **Access List** ‚Äî URL becomes `?tab=access-list`
6. Click **General** ‚Äî URL returns to `?tab=general`

## Steps ‚Äî page refresh preserves tab

7. With URL at `/workflow/<slug>/config?tab=api-keys`, press **F5** (page refresh)
8. Verify the page loads directly on the **API Keys** tab (not the General tab)
9. Refresh at `?tab=variables` ‚Äî Variables tab is selected on load
10. Refresh at `?tab=access-list` ‚Äî Access List tab is selected

## Steps ‚Äî invalid tab param fallback

11. Navigate to `/workflow/<slug>/config?tab=nonexistent`
12. Verify the page falls back to the **general** tab (invalid tab IDs default to "general")

## Steps ‚Äî no tab param = general tab

13. Navigate to `/workflow/<slug>/config` (no `?tab=`)
14. Verify the General tab is active (default: `initialTab ?? "general"`)

## Steps ‚Äî browser Back/Forward

15. Click through several tabs (URL updates each time)
16. Press the browser Back button ‚Äî verify the previous tab is restored (history entries created by `router.replace`)

## Expected result
- Tab click: URL updated to `?tab=<tab-id>` via `router.replace` (no scroll jump)
- Page load with `?tab=<id>`: correct tab pre-selected (server reads `searchParams.tab`)
- Invalid `?tab=` value: falls back to `general`
- Missing `?tab=`: falls back to `general`
- Valid tab IDs: `general`, `api-keys`, `access-list`, `variables`, `provider-keys`, `executions`, `credentials`

## Failure indicators
- Clicking tabs does not update the URL
- Refreshing at `?tab=api-keys` lands on the General tab instead
- Invalid `?tab=notreal` causes a 404 or error page instead of graceful fallback to general

## Severity rationale
Medium: URL-based tab state allows users to bookmark specific config views and share links with teammates; without it, every page load requires manual re-navigation.
